**Buy anything from any merchant with one API call — a step an agent or app
drops into a workflow. Lane handles the rest.**

The Order API is the **programmatic** way to use Lane: your agent or backend
calls it as part of an automated flow, with no human in the loop per order. You
send a plain-language instruction plus where to ship it. Lane runs the full
checkout on our servers — find the product, add to cart, sign in if needed, pay,
confirm — and reports back when the order is placed. No browser automation, no
merchant integrations, and no payment handling on your side.

## The integration in one picture

Two parties: the **Agent Platform** (your agent or app) passes the order details
to **Lane** over the API, and Lane runs the checkout at the merchant on your
behalf.

```
Agent Platform                   Lane                         Merchant
--------------                   ----                         --------
  POST /agent/v1/orders          |                            |
  { instruction, ship_to, ... }  |                            |
  +----------------------------->|                            |
        202 { order_id }         |                            |
  <-----------------------------+|                            |
        |                        |  finds product, adds to     |
        |                        |  cart, signs in, pays,      |
        |                        |  confirms                   |
        |                        +--------------------------->|
        |                        |<---------------------------+
        |  webhook: order.completed                            |
  <-----------------------------+|                            |
        |  { order_number, amount_charged, tracking }          |
        v                        |                            |
```

Everything between the `202` and the `order.completed` webhook happens on Lane's
servers. A typical order takes a few minutes. Batching versus one-at-a-time makes
no difference on our side, and there is no cold-start penalty.

## How it relates to the plugin

The Order API is the **headless** way to use Lane — for an agent or app running
on its own. Where the [Lane Plugin](/plugin/installation) is interactive (an
agent and a person sharing a surface, the person approving each purchase with a
passkey), the Order API is pre-authorized: you set a payment method and a
spending authorization once, and every order after that is a single
server-to-server call your workflow makes with no per-order approval.

<Note>
  Under the hood, an order is one merchant checkout — a *mandate* — and your
  account's spending authorization is the *intent*-level budget. You don't need
  that vocabulary to use the API, but if you're also using the plugin, the
  [shared model](/concepts/model) explains how the two line up.
</Note>
